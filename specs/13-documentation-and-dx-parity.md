# Documentation and DX Parity

## Purpose

Neuron must be teachable.

Documentation is part of core completeness because the architecture is only valuable if users can reliably compose systems out of it.

## Requirements

- A root README that describes the redesign and quickstart.
- A crate map that explains what each crate does.
- A small progressive tutorial series that builds up to multi-agent composition.
- Clear guidance on mock vs real path testing.

## Current Status

Root README, crate map, and ARCHITECTURE.md are in place.
Progressive tutorial series is underway in `docs/book/`.

Still required:

- examples that match docs
- clear guidance on mock vs real path testing
