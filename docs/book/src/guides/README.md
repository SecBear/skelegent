# Guides

Practical guides for working with each subsystem in skelegent:

- **[Providers](providers.md)** -- How to configure and use LLM providers (Anthropic, OpenAI, Ollama).
- **[Tools](tools.md)** -- How to create tools, register them, and use them with operators.
- **[Operators](operators.md)** -- ReAct (reasoning loop with tools) and single-shot (one call, no tools) operators.
- **[State](state.md)** -- Persisting data with in-memory and filesystem state stores.
- **[Orchestration](orchestration.md)** -- Multi-agent composition with the local orchestrator.
- **[Hooks](hooks.md)** -- Observing and intervening in operator execution.
- **[MCP](mcp.md)** -- Model Context Protocol integration.
- **[Testing](testing.md)** -- Testing patterns, mock providers, and the test-utils feature.

Each guide focuses on one subsystem. For architectural context (why things are designed this way), see the [Architecture](../architecture/README.md) section.
